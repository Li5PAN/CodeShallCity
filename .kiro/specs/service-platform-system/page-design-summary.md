## 设计内容概览

### 一、服务提供者端(PROVIDER)设计

#### 1. 创作中心页面 (Creator Center)
- **路由**: `/creator`
- **功能**: 
  - 三个标签页: 文章管理、需求管理、服务管理
  - 服务发布表单(标题、类别、价格、描述、封面、周期)
  - 服务列表展示(状态筛选、类别筛选、搜索)
  - 服务卡片操作(编辑、删除、查看详情)
- **状态管理**: 待审批、已上架、已拒绝、已下架
- **表单验证**: 完整的字段验证规则

#### 2. 个人中心 - 卖家订单页面
- **路由**: `/profile` (seller-orders标签页)
- **功能**:
  - 订单筛选(状态、日期范围、搜索)
  - 统计卡片(待处理、进行中、已完成、已失败)
  - 订单状态更新对话框
  - 状态转换规则(待处理→进行中→已完成/已失败)
- **操作**: 查看详情、更新状态、联系买家

#### 3. 服务详情页面
- **路由**: `/services/{id}` (弹窗模式)
- **展示内容**: 封面、标题、价格、类别、评分、状态、提供者信息、服务描述
- **权限控制**: 
  - 所有用户可查看已上架服务
  - 所有者可编辑删除
  - 管理员可管理所有服务

### 二、管理员端(ADMIN)设计

#### 1. 管理员仪表板 (Dashboard)
- **路由**: `/admin/dashboard`
- **功能**:
  - 时间范围选择(最近7天/30天/90天/自定义)
  - 关键指标卡片(总用户数、活跃用户、总订单数、总收入)
  - 订单趋势图表(折线图)
  - 订单状态分布统计
  - 服务类别排名(横向条形图)
  - 待处理事项提醒
- **数据刷新**: 每5分钟自动刷新

#### 2. 用户管理页面
- **路由**: `/admin/users`
- **功能**:
  - 搜索和筛选(用户名/邮箱、角色、状态)
  - 用户列表表格(ID、头像、用户名、邮箱、角色、状态、注册时间)
  - 用户详情弹窗(基本信息、活动统计、操作历史)
  - 用户编辑弹窗(修改信息、角色、状态)
- **操作**: 查看、编辑、暂停/激活、更改角色、重置密码、删除

#### 3. 服务审批页面
- **路由**: `/admin/services`
- **功能**:
  - 筛选器(状态、类别、搜索)
  - 待审批服务列表
  - 服务详情查看弹窗
  - 审批操作(批准/拒绝)
- **审批流程**: 
  - 批准: 服务上架,通知提供者
  - 拒绝: 填写拒绝原因,通知提供者
- **审核标准**: 描述完整性、价格合理性、平台规范、违规内容

#### 4. 申诉管理页面
- **路由**: `/admin/appeals`
- **功能**:
  - 筛选器(状态、类型、订单号搜索)
  - 申诉列表展示
  - 申诉详情处理弹窗(订单信息、交易双方、申诉信息、支持材料)
  - 处理决定(申诉成功/失败)
- **处理流程**:
  - 申诉成功: 订单标记为失败,通知双方
  - 申诉失败: 维持订单状态,通知双方
- **记录**: 所有操作记录审计日志

#### 5. 认证审核页面
- **路由**: `/admin/certifications`
- **功能**:
  - 筛选器(状态、申请人搜索)
  - 认证申请列表
  - 认证详情审核弹窗(申请人信息、认证资质材料、历史活动)
  - 审核决定(批准/拒绝)
- **审核流程**:
  - 批准: 用户角色升级为PROVIDER
  - 拒绝: 填写拒绝原因,通知申请人
- **审核标准**: 身份真实性、技能资质、平台活动、信誉评估

#### 6. 内容审核页面
- **路由**: `/admin/content`
- **功能**:
  - 三个标签页: 文章审核、需求审核、评论审核
  - 筛选和排序(类别、排序方式、搜索)
  - 内容操作(查看、编辑、删除、置顶、推荐)
  - 删除确认对话框(删除原因、详细说明、通知作者)
- **删除流程**: 选择原因→填写说明→确认删除→通知作者→记录日志

#### 7. 审计日志页面
- **路由**: `/admin/audit-logs`
- **功能**:
  - 筛选器(操作类型、操作人、日期范围、搜索)
  - 日志列表表格(时间、操作人、操作类型、目标、IP地址、详情)
  - 日志详情弹窗(基本信息、操作目标、操作详情JSON)
  - 日志导出功能(CSV/Excel)
- **日志内容**: 用户认证、管理员操作、订单交易、申诉处理
- **保留策略**: 至少90天,重要日志永久保留

### 三、业务流程图

#### 1. 服务提供者业务流程
```
登录 → 申请认证 → 管理员审核 → 角色升级 → 发布服务 → 
服务审批 → 服务上架 → 用户购买 → 订单管理 → 更新状态 → 
订单完成 → (可能)申诉处理
```

#### 2. 管理员审核流程
```
登录后台 → 选择功能(认证审核/服务审批/申诉管理/用户管理/内容审核) → 
查看详情 → 做出决定 → 通知相关方 → 记录审计日志
```

#### 3. 订单申诉流程
```
订单完成 → 买家不满意 → 提交申诉 → 填写表单 → 上传材料 → 
管理员审查 → 处理决定 → 通知双方 → 记录日志
```
